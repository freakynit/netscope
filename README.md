# NetScope

NetScope is an early macOS-focused Rust network inspection core. It starts with an explicit loopback HTTP/1.1 proxy and passive libpcap capture; it does not install transparent system routing or a Network Extension.

Only inspect traffic you are authorized to inspect.

## Features

- CLI proxy on `127.0.0.1:8080` by default, HTTP/1.1 request/response inspection, persistent connections, fixed-length and chunked streaming, raw CONNECT tunnels, and WebSocket upgrade forwarding.
- Optional HTTPS MITM using a locally generated CA, per-host leaf certificates, and upstream leaf-certificate metadata (subject, issuer, serial, expiry).
- Passive capture on one or more interfaces, capture-time BPF prefiltering, Ethernet IPv4/IPv6 TCP/UDP/ICMP decoding, TCP metadata/options, fragmentation metadata, DNS answers/CNAMEs/latency, TLS ClientHello SNI/ALPN/version/cipher offers, and bounded in-order TCP reassembly.
- Structured `TrafficEvent` records in NDJSON plus an in-process broadcast store. Packet events carry canonical bidirectional connection IDs, direction when it can be inferred, wire/captured/payload lengths, and packet metadata.
- Optional rotating `pcapng` raw-frame retention, preserving libpcap's link type. Packet events point back to the exact Enhanced Packet Block by packet ID, filename, and byte offset.
- A versioned local WebSocket hook protocol with active `continue`, `modify`, `drop`, `respond`, and `delay` actions, plus non-blocking observers.
- Modular Rust boundaries for capture, proxying, protocol parsing, certificates, hooks, and storage. Swift is not in the data path.

## Build

macOS supplies `libpcap` and the `security` utility.

```sh
cargo build --release
./target/release/netscope --help
./target/release/netscope interfaces
```

If libpcap headers are unavailable, build proxy-only support:

```sh
cargo build --release --no-default-features
```

## Certificates and HTTPS inspection

Generate a CA; its private key is stored owner-only in the selected directory:

```sh
./target/release/netscope cert generate --ca-dir ~/.netscope
sudo ./target/release/netscope cert install --ca-dir ~/.netscope
```

Remove its System Keychain trust and local certificate/key files:

```sh
sudo ./target/release/netscope cert remove --ca-dir ~/.netscope
```

Start an inspecting proxy and configure a browser/client to use `127.0.0.1:8080` for both HTTP and HTTPS:

```sh
./target/release/netscope proxy --mitm --ca-dir ~/.netscope --events traffic.ndjson
curl -x http://127.0.0.1:8080 https://example.com/
```

Without `--mitm`, CONNECT is a byte-for-byte tunnel and HTTPS contents are not visible.

## Local hooks and examples

Start the proxy with a loopback WebSocket hook endpoint. The default interceptor deadline is 250 ms; JSON body events are bounded to 1 MiB by default.

```sh
./target/release/netscope proxy --hook-listen 127.0.0.1:8765 --hook-timeout-ms 500
python3 -m pip install websockets
python3 examples/hooks/python/hook.py log
curl -x http://127.0.0.1:8080 http://httpbin.org/get
```

Node.js uses the same wire protocol:

```sh
npm install ws
node examples/hooks/node/hook.js request-header
curl -i -x http://127.0.0.1:8080 http://httpbin.org/headers
```

The runnable Python and Node examples cover logging, headers, JSON rewrite, blocking, synthetic responses, delays, body saving, and non-blocking observation. See [examples/hooks/README.md](examples/hooks/README.md) and the stable [hook protocol](docs/HOOK_PROTOCOL.md).

## Passive capture

Capture normally requires root or an appropriate capture entitlement:

```sh
./target/release/netscope interfaces
sudo ./target/release/netscope capture --interface en0 --interface lo0 --protocol tcp --port 443 --events capture.ndjson --pcapng capture.pcapng --pcapng-rotate-mb 256
tail -f capture.ndjson | jq .
```

`length` remains the transport payload length for compatibility. New packet fields are explicit: `wire_length`, `captured_length`, `payload_length`, `ip_version`, `ttl_or_hop_limit`, `fragmentation`, `tcp_flags`, `sequence_number`, `acknowledgement_number`, `window_size`, `tcp_options`, `udp_length`, and `icmp_type`/`icmp_code`. TCP anomaly attributes are distinct: `duplicate_ack`, `retransmitted_segment`, `out_of_order`, and `zero_window_probe`. `raw_packet` links an event to the corresponding pcapng block. With more than one interface, NetScope inserts the interface name into the pcapng filename.

## Current limitations

- Certificate pinning, custom trust stores, and applications with their own CA bundles will reject MITM. NetScope does not bypass them.
- HTTP/2, HTTP/3/QUIC, encrypted DNS, generic UDP flow/reassembly, and UDP interception are not implemented.
- Passive TCP reassembly is in-order and bounded to a 128 KiB prefix; it does not reorder out-of-order segments or retain a full flow history. Passive TLS decoding extracts ClientHello metadata and a leaf certificate only from unencrypted TLS 1.0-1.2 Certificate handshakes; TLS 1.3 certificate metadata requires the explicit MITM proxy. Passive HTTP transaction parsing is not implemented.
- Hostname filtering only applies when DNS questions or TLS SNI are observed; there is no hostname-to-flow cache. Process attribution is not implemented.
- HTTP compression transforms and WebSocket frames are forwarded, but not parsed or modified. JSON hook bodies are only fixed-length, uncompressed JSON within the configured bound; chunked/non-JSON/oversized bodies stay streamed.
- HTTPS MITM has no leaf cache and reports basic upstream leaf metadata, not full chain analysis.
- The hook WebSocket is local-only and unauthenticated. A timed-out, crashed, or invalid active hook fails open; observers are best effort.
